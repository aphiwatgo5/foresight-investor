Obsidian

What it is: A local markdown knowledge-base app; in this pattern it's the human's
"IDE" for browsing and navigating the LLM-maintained wiki.

Summary

Obsidian renders the wiki's interlinked markdown so the human can follow links,
inspect the graph, and read updated pages in real time while the LLM edits them. The
division of labor: Obsidian is the IDE, the LLM is the programmer, the wiki is the codebase
(the-llm-wiki-pattern).

Key facts

  • Reads plain markdown files on disk — the wiki is just a folder of .md files. (the-llm-wiki-pattern)
  • Graph view is the best way to see the wiki's shape: hubs, clusters, orphans. (the-llm-wiki-pattern)
  • Web Clipper browser extension converts web articles to markdown for raw/. (the-llm-wiki-pattern)
  • Dataview plugin queries page YAML frontmatter to build dynamic tables/lists. (the-llm-wiki-pattern)
  • Marp plugin renders markdown into slide decks. (the-llm-wiki-pattern)
  • Attachment-download settings/hotkeys can pull article images locally into raw/assets/. (the-llm-wiki-pattern)

Relationships

  • Hosts and visualizes this wiki, including overview and index.
  • Supports the ingest-query-lint loop on the human's side (browsing, graph inspection).

Notes & tensions

  • This vault already has .obsidian/ present, confirming Obsidian is the active front end.
  • LLMs can't read markdown with inline images in one pass — read text first, then view
    referenced images separately. (the-llm-wiki-pattern)
