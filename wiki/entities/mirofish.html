MiroFish

What it is: An open-source multi-agent "swarm intelligence" prediction engine that
constructs a high-fidelity digital world of thousands of LLM agents to rehearse and predict
how real-world events might unfold.

Summary

MiroFish ingests "seed" information from reality (news, policy drafts, financial signals,
even fiction) and builds a parallel digital sandbox populated by autonomous agents that have
personality, long-term memory, and behavioral logic. Those agents interact and socially
evolve, and the user injects variables from an outside ("God's-eye") view to deduce future
trajectories (mirofish source). It serves both serious decision rehearsal and playful
creative simulation.

Key facts

  • Powered by multi-agent technology; core simulation engine is oasis (by camel-ai). (mirofish)
  • Uses zep for agent long-term memory; supports any OpenAI-SDK-compatible LLM, recommending Alibaba Qwen-plus. (mirofish)
  • Five-stage workflow: Graph Building → Environment Setup → Simulation → Report Generation → Deep Interaction. (mirofish)
  • Stack: Node 18+ frontend (:3000) + Python 3.11–3.12 backend (:5001), managed with uv; Docker option. (mirofish)
  • Author: 666ghj; strategically supported/incubated by shanda-group. (mirofish)
  • Repo: github.com/666ghj/MiroFish; live demo at 666ghj.github.io/mirofish-demo. (mirofish)

Relationships

  • Built on oasis; memory via zep; backed by shanda-group; created by 666ghj.
  • Implements multi-agent-simulation, produces collective emergence, uses graphrag.
  • Practical setup & use cases: mirofish-how-to-use-and-applications.

Notes & tensions

  • Strong predictive claims ("predicting anything") with no stated accuracy evaluation. (mirofish)
  • Flagged as "high consumption" — suggests keeping early simulations under 40 rounds; true cost unstated. (mirofish)
