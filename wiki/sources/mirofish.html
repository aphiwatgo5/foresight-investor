MiroFish (GitHub README)

One-line: README for mirofish, an open-source multi-agent "swarm intelligence"
engine that builds a high-fidelity digital world of thousands of LLM agents to rehearse and
predict future trajectories from real-world seed information.

Key takeaways

  • mirofish turns seed inputs (news, policy drafts, financial signals, even novels) into
    a parallel digital world where thousands of agents with personality, long-term memory, and
    behavioral logic interact and socially evolve. (mirofish)
  • Users inject variables from a "God's-eye view" to deduce future trajectories — framed as
    rehearse the future in a digital sandbox. (multi-agent-simulation)
  • Two use framings: macro = zero-risk rehearsal lab for decision-makers (policy/PR
    testing); micro = creative sandbox (e.g. deducing the lost ending of Dream of the
    Red Chamber    ).
  • Five-stage workflow: Graph Building (graphrag) → Environment Setup (persona/agent
    config) → Simulation (dual-platform parallel) → Report Generation (ReportAgent) → Deep
    Interaction (chat with any agent).
  • Built on oasis (by camel-ai) for simulation; uses zep for agent memory;
    recommends Alibaba Qwen-plus as the LLM. Incubated by shanda-group. Author: 666ghj.

Summary

MiroFish positions itself as a next-generation AI prediction engine powered by multi-agent
technology (multi-agent-simulation). Its central mechanism: extract "seed" information
from the real world, then automatically construct a high-fidelity parallel digital world
populated by many autonomous agents. Those agents — each with independent personality,
long-term memory, and behavioral logic — freely interact and undergo social evolution,
producing collective emergence (see swarm-intelligence). The human acts as an outside
operator, injecting variables to precisely deduce how events might unfold.

The product targets two audiences. At the macro level it's a "rehearsal laboratory" letting
decision-makers test policies and PR at zero real-world risk. At the micro level it's a
playful creative sandbox — the README cites simulating university public-opinion events and
predicting the lost ending of the classic novel Dream of the Red Chamber from its first 80
chapters. The pitch: let every "what if" see its outcome.

The five-stage pipeline runs from seed extraction and GraphRAG construction (graphrag),
through entity/relationship extraction and persona generation, to dual-platform parallel
simulation with dynamic temporal memory updates, a ReportAgent that produces a prediction
report, and a final deep-interaction layer where the user can chat with any agent in the
simulated world. Technically it is a Node (18+) frontend plus a Python (3.11–3.12) backend
managed with uv, runnable from source (npm run dev, frontend :3000 / backend :5001) or
via Docker. It supports any OpenAI-SDK-compatible LLM, recommending Alibaba's Qwen-plus, and
uses zep for memory.

Connections

  • The engine under the hood: oasis (open-source, by camel-ai).
  • Memory layer: zep.
  • Core methods: multi-agent-simulation, swarm-intelligence, graphrag.
  • Backed by: shanda-group; authored by 666ghj.

Open questions

  • How faithful are the predictions? The README makes strong claims but cites no accuracy
    evaluation or benchmark.
  • What is the actual cost profile? It warns of "high consumption" and suggests <40 simulation
    rounds — order-of-magnitude token cost is unstated.
  • How does "dual-platform parallel simulation" work concretely, and what are the two platforms?
  • Screenshots and demo videos (Wuhan University opinion sim; Dream of the Red Chamber) are
    referenced but not captured locally — worth archiving if they matter.
