Wiki as a compounding artifact

Definition: The core thesis of this system — knowledge is compiled once at ingest and
kept current, rather than rediscovered from scratch on every query (as in RAG).

Summary

In standard RAG, every question re-retrieves chunks and re-derives an answer; nothing
accumulates. Here, the LLM integrates each new source into a persistent, interlinked wiki.
Because cross-references already exist, contradictions are already flagged, and the
synthesis already reflects everything read, the wiki gets richer with every source and
every question (the-llm-wiki-pattern).

Key facts

  • Contrast: RAG = retrieve-and-re-derive per query; this = compile-once, maintain-current. (the-llm-wiki-pattern)
  • The compounding works because maintenance/bookkeeping cost is near-zero for an LLM. (the-llm-wiki-pattern)
  • Good query answers can be filed back as synthesis pages, so explorations compound too. (the-llm-wiki-pattern)

Relationships
