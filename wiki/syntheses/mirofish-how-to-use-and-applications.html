MiroFish — How to use & applications

Bottom line: MiroFish is a "rehearse-the-future" sandbox. You install it, plug in two API
keys (an LLM + a memory service), feed it a real-world situation, and it runs a world of
thousands of AI agents to produce a prediction report you can interrogate. Below: how to set
it up and run it, and what it's actually good for. (All from mirofish; claims are the
project's own.)

Part 1 — How to use it

A. What you need first (prerequisites)

Tool Version Why
Node.js 18+ Frontend (includes npm)
Python 3.11–3.12 Backend
uv latest Python package manager

Plus two API keys:
- LLM key — any OpenAI-SDK-compatible API. Project recommends Alibaba Qwen-plus via
the Bailian platform. (zep is separate — see below.)
- Zep key — zep cloud, for the agents' long-term memory. Free monthly quota is
enough for simple use.

B. Install & start (one-time setup)

  1. Copy the config template: cp .env.example .env
  2. Edit .env and fill in the keys:
    ```
    # Option A — Alibaba Qwen (DashScope)
    LLM_API_KEY=your_dashscope_api_key
    LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
    LLM_MODEL_NAME=qwen-plus

# Option B — Google Gemini
LLM_API_KEY=your_gemini_api_key
LLM_BASE_URL=https://generativelanguage.googleapis.com/v1beta/openai/
LLM_MODEL_NAME=gemini-2.5-flash

# Zep (agent long-term memory)
ZEP_API_KEY=your_zep_api_key
`` 3. Install everything:npm run setup:all4. Start it:npm run dev- Frontend →http://localhost:3000- Backend API →http://localhost:50015. **Or** with Docker:cp .env.example .envthendocker compose up -d` (maps ports 3000/5001).

C. How you actually use it (the day-to-day flow)

You only do two things; MiroFish does the rest:
1. Upload "seed" materials — a data/analysis report, a news item, a policy draft, or even
a story.
2. Describe what you want to predict in plain natural language.

Then MiroFish runs its 5-stage pipeline (multi-agent-simulation):
1. Graph Building — extracts the seed info and builds a knowledge graph (graphrag).
2. Environment Setup — generates agent personas and relationships.
3. Simulation — thousands of agents interact over simulated time (swarm-intelligence).
4. Report Generation — a ReportAgent writes up a detailed prediction report.
5. Deep Interaction — you can chat with any agent in the simulated world, or with the
ReportAgent, and inject "what if" variables from a god's-eye view.

Cost tip: the project warns it's "high consumption." Start with simulations of fewer
than 40 rounds before scaling up. (mirofish)

Part 2 — Applications (what it's good for)

MiroFish frames two levels of use:

Macro — a rehearsal lab for decision-makers

Test high-stakes decisions in a zero-risk virtual world before acting for real:
- Public-opinion / PR simulation — e.g. the project's demo simulating a university
public-opinion event (Wuhan University). Rehearse how an announcement or crisis would land.
- Policy testing — try a policy draft and see how a simulated population reacts.
- Financial / market prediction — feed financial signals and deduce trajectories
(listed as a coming example).

Micro — a creative sandbox for individuals

Playful "what if" exploration:
- Story / narrative endings — the demo predicts the lost ending of Dream of the Red
Chamber from its first 80 chapters.
- Imaginative scenarios — explore any hypothetical and watch it play out.

The unifying idea

From serious forecasting to playful simulation, MiroFish's pitch is to let every "what if"
see its outcome — rehearse the future in a digital sandbox, win decisions after countless
simulations.
