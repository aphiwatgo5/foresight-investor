Security Sourcing Pipeline — v4.0 workflow

Bottom line: The day-to-day workflow that turns the quantitative signal engine into investment
decisions. Replaces v3.4's "evaluate one name at a time" approach with a daily universe screen →
shortlist → MDAEM verdict flow that scales across all 280 instruments. For the operator's own
book; not advice.

The 5-step workflow

Step 1 — Universe screen (automated, daily)

scripts/signals.py ranks every instrument across the 4 sleeve composites; the top decile per
sleeve (top ~28 names) is flagged for review. Output: scripts/_hist/sleeves.json (live percentile
scores). The dashboard's Strategy Sleeves tab surfaces these visually.

What changes from v3.4: instead of the operator deciding which names to evaluate, the screen
surfaces candidates by quantifiable edge. The operator's judgment moves to Step 2.

Step 2 — MDAEM v4.0 pass (semi-automated)

For each flagged candidate, run scripts/mdaem_v4.py → returns quant Dim 3/5/6/7/9 scores +
composite_quant_only + tier. Then complete the 5 qualitative dims manually (Quality, Fin
Health, Earn Quality, Catalysts, Asymmetry) — these need judgment.

Apply the 10 hard gates unchanged from v3.4:
- Gate 1 Asymmetry (bull:bear ≥1.5:1) · Gate 2 Catalyst (<36mo) · Gate 3 Liquidity ·
Gate 4 Accounting · Gate 5 Bear case articulable · Gate 6 SA-Quant (not Sell) ·
Gate 7 Revisions (≥0.2) · Gate 8 M&A ceiling · Gate 9 Multiple (<50x sector) ·
Gate 10 Financing (runway past catalyst).

Any gate fail → REJECT (documented reason, mapped to a Ref).

Step 3 — Style fit (which sleeve?)

Tag the name with its best-fit sleeve (a name can be in 2):
- Quality — high margins, low debt, stable growth (V, MA, NVDA, LLY, TSM, ISRG).
- Momentum — strong relative strength, SMA golden cross, RSI 50-70 (the AI/cyber trade).
- Macro-rotation — regime-conditioned fit (high-beta in bull; low-vol in bear).
- Value/contrarian — beaten-down with a revisions/momentum inflection (CVS, UNH, BABA).

Sizing is constrained by the sleeve's target weight (strategy-sleeves) AND the meta-force caps
in portfolio-blueprint.

Step 4 — Portfolio construction (the barbell + sleeves)

  • Check portfolio-blueprint concentration caps: no ticker >8%, AI-capex cluster ≤25%, cash ≥3%.
  • Confirm the position fits the regime-monitor stance (e.g., new momentum adds paused in bear).
  • Confirm theme-linkage-map correlation budget not breached.

Step 5 — File the decision + accountability

  • Add a row to decision-register (D-series): ticker · action · conv · target wt · entry · invalidation.
  • File a falsifiable P-call in prediction-ledger (success/failure thresholds + resolve-by date).
  • Open a D-series row in track-record for mark-to-market.
  • Log the decision in log.md with op decide.

The monthly review (BACKTEST op)

At each monthly SCAN per CADENCE.md:
1. Re-run scripts/signals.py + scripts/mdaem_v4.py on fresh data.
2. Score last month's decisions in track-record — hit rate by conviction + by sleeve.
3. Re-balance sleeve weights if any sleeve is >5pts off target.
4. Score any matured P-calls.
5. Surface new candidates from the daily screen that weren't yet reviewed.

The quarterly backtest refresh (BACKTEST op)

  • Re-run scripts/fetch_history.py --inc → scripts/signals.py → scripts/backtest.py.
  • Compare this quarter's sleeve metrics to last; flag any decay.
  • Tune ensemble weights (e.g., if momentum Sharpe drops, raise Quality weight).

How this differs from v3.4 (the upgrade in one paragraph)

v3.4 was manual name-by-name — the operator heard about a name, ran MDAEM, decided. v4.0 is
screen-first — the quant layer surfaces the top decile daily, the operator runs MDAEM only on
the shortlist, and every decision carries a sleeve tag + measured historical alpha. The gates,
weights, refinements, and accountability are all unchanged. Net effect: the same decision
quality with 10× the coverage and a built-in "does this style actually work?" loop.
