Operating Cadence

The rhythm that keeps this engine from going stale. The 2026-06-28 burst followed by a 16-day
silence is the failure mode this doc exists to prevent. Pin it; follow it.

Insight & decisions for the operator's own book — not personalized advice to others. See
CLAUDE.md for the full schema; this file defines when each operation runs.

The three rhythms

1. MONITOR — weekly (every weekend, ~Sunday)

The lightest touch. Catches regime shifts before they become surprises.

🤖 Auto (Sunday 06:00, no human needed): a macOS launchd job runs
scripts/weekly_refresh.sh, which chains the three refresh scripts:
1. scripts/refresh.py --macro — pulls live macro/sentiment indicators → _snapshot.json
2. scripts/refresh_instruments.py — pulls data for all ~250 instruments → _instruments.json
3. scripts/build_instrument_tables.py — regenerates instrument-universe + the per-theme
instrument tables on all 30 concept pages.

The job auto-commits the result (no push — publishing is manual via scripts/deploy_web.sh).
Logs at scripts/_logs/cron.log. Install/uninstall commands in
scripts/com.foresight-investor.weekly-refresh.plist.

Then the human/agent MONITOR pass (review what the auto-refresh produced):
- Read _snapshot.json + signal-monitor — flag anything that crossed a warn/flip threshold.
- If a threshold flipped → escalate to that theme page + regime-monitor; note in the
decision-register if it triggers an action.
- Check prediction-ledger for any call whose resolve-by passed → score it (✅/❌/◐).
- Append a ## [date] monitor | <headline> line to log.md; refresh hotcache.md.

Time budget: ~15 min of review (the data pull is now automated). This is the floor — never
skip two weeks in a row.

2. SCAN — monthly (first weekend of the month)

The radar refresh. Full trend-radar + watchlist pass:
- Re-score conviction/stage on every theme (or at least the active clusters); note what moved & why.
- Adjust watchlist (add/remove names; update entry triggers + invalidations as data shifts).
- Update frontier-watch — promote any emerging theme whose score now justifies a concept page.
- Reconcile decision-register realized/exit decisions into track-record; compute rolling
hit-rate + alpha vs VOO.
- Refresh macro-regime and theme-linkage-map reads.
- Append ## [date] scan | <headline> to log.md; refresh hotcache.md + index.md.

Time budget: ~1–2 hours.

3. REVIEW — quarterly (and the annual)

The accountability + recalibration cycle.
- Score all matured predictions in prediction-ledger; update the calibration read
(do conv-4 calls hit more than conv-2/3?).
- Full track-record close-out for the quarter: realized P&L, win-rate, alpha, worst drawdown.
- Revisit the portfolio-blueprint target allocations; rebalance logic check.
- If a bubble-watch is warranted → run BUBBLE-WATCH (CLAUDE.md §10) on the current target.
- Propose schema/framework refinements (the MDAEM-style "lessons" loop, applied vault-wide).
- Append ## [date] review | <quarter> close to log.md.

Time budget: ~2–4 hours.

Ad-hoc operations (on demand, not scheduled)

Op When to run What it produces
INGEST Human adds a source to raw/ New wiki/sources/ + entity/concept pages (CLAUDE.md §5)
QUERY Human asks a question Cited answer; good ones filed as wiki/syntheses/
DECIDE A signal flips or a name crosses an entry/invalidation New/updated row in decision-register + a P-call in prediction-ledger + track-record entry
BUBBLE-WATCH Macro feels frothy or a quarterly trigger hits wiki/syntheses/bubble-risk-<target>-<date>.md (CLAUDE.md §10)
DEEP-DIVE A theme/security needs a refresh between scans Updated theme page + security card(s)
LINT Roughly monthly, or when link drift is suspected Health-check pass; fixes applied after human confirms

The staleness contract

  • Hard rule: the gap between any two log.md entries must not exceed 14 days. If it does,
    run at minimum a MONITOR pass before doing anything else — staleness is the engine's #1 enemy.
  • Every page's frontmatter last_scan is the source of truth for its age. If a page is older than
    its rhythm allows, it carries a stale flag until refreshed.
  • signal-monitor and regime-monitor are the most time-sensitive — refresh weekly. Theme
    pages and the radar tolerate monthly. The prediction ledger is event-driven (score on resolve).

How to know you're behind

Run:

grep "^## \[" log.md | tail -5          # last 5 ops — if oldest is >14d, you're behind
grep -l "last_scan: 202" wiki/syntheses/*.md | xargs grep "last_scan"  # scan dates
python3 scripts/refresh.py --check      # drift report vs last snapshot
