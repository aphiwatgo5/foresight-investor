🧱 How it's built — three layers (this is how you trace anything)

  1. raw/ — Source of truth. Immutable original documents (articles, frameworks, notes).
    Never edited. Every claim in the wiki traces back to a file here (or a dated web lookup).
  2. wiki/ — Generated knowledge, owned entirely by the agent:
    - wiki/sources/ — one summary per ingested source
    - wiki/entities/ — people, orgs, products, tools, ETFs, securities
    - wiki/concepts/ — ideas, methods, and investment themes
    - wiki/syntheses/ — cross-source analysis, comparisons, answers worth keeping
  3. Schema & navigation — CLAUDE.md (operating manual) · index.md (catalog) · log.md
    (timeline) · hotcache.md (rolling state) · this home.md.

Citations tell you the provenance of every line: (source-page) = from a source ·
(web: …, date) = web lookup · (inference) = the agent's reasoning · (fact)/(est.) on figures.

🛠️ Features & skills (everything the engine can do)

Core wiki operations (CLAUDE.md §5)

  • Ingest — read a new source → summarize, create/update ~10–15 entity & concept pages,
    cross-link, flag contradictions, log.
  • Query — answer questions from the wiki with citations; file good answers as durable syntheses.
  • Lint / health-check — scan for contradictions, stale claims, orphans, dead links, missing
    cross-references; propose fixes before applying.

Investing engine — the "Futurist Investor" (CLAUDE.md §10; cadence CADENCE.md)

  • Scan (monthly) — radar refresh: update conviction & stage, adjust the watchlist, score
    matured predictions.
  • Monitor (weekly, v2) — light sensor sweep via signal-monitor: catch regime shifts
    between scans; score any matured P-call. Powered by scripts/refresh.py.
  • Decide (on demand, v2) — issue/exit sized calls (ADD/TRIM/HOLD/EXIT) in decision-register
    for the operator's own book, via the mdaem-framework methodology + portfolio-blueprint caps.
    Each decision filed as a falsifiable P-call + scored in track-record.
  • Refresh (v2) — run scripts/refresh.py to pull live data (prices/CAPE/F&G/yields/fundamentals)
    with full provenance; feeds signal-monitor, theme pages, and the dashboard.
  • Bubble-watch — on-demand bubble-risk analysis of any target from live-verified data run
    through 7 frameworks. First run: bubble-risk-us-equity-2026-06-28.
  • The radar — 7 meta-forces → 30 themes, each an analyst-grade page. Master table: trend-radar.
  • Living hubs — trend-radar · watchlist · prediction-ledger (the original three) +
    v2: signal-monitor · decision-register · portfolio-blueprint · regime-monitor ·
    track-record · frontier-watch    .
  • Single-name methodology — mdaem-framework (10 gates / 10 dimensions) + seeking-alpha-quant
    calibration; promoted in v2 from external-only into the vault's decision-register.

The dashboard

  • outlook-dashboard.html — a single, offline, interactive dashboard fusing all the above:
    bubble frameworks, the 30-theme radar (sortable/filterable), quarterly indicator charts, and a
    Features & Map tab. Edit its one DATA block (via Claude Code or scripts/refresh.py) to
    refresh; no online link. (Signal/Decision/Portfolio/Regime tabs are the next enhancement.)
